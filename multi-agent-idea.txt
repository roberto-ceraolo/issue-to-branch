⏺ You're right to push back. opencode is much more capable here than I assumed.
  Here's what it actually supports:

  opencode serve — starts a headless HTTP server (no TUI)

  opencode run --attach http://localhost:PORT --session <id> --format json 
  "message" — send a follow-up message to an existing session and get structured
   JSON events back

  opencode attach <url> — connect to a running server

  This means the real architecture is:

  For each sub-agent:
    1. opencode serve --port 4001 --dir ./worktree-task-A   (headless,
  background)
    2. opencode run --attach http://localhost:4001 --format json "implement
  X..."
       → streams JSON events back to me
    3. If the agent's response contains a question or blocker:
       → I relay it to you, or answer it myself
       → opencode run --attach ... --session <same-id> "here's the answer,
  continue"
    4. When done: push branch, gh pr create

  So it's genuinely interactive and stateful — I can have a multi-turn dialogue
  with each sub-agent, relay questions up to you when needed, and drive them to
  completion.

  The sessions are independent per worktree/port, so agents don't interfere with
   each other.
